Installing 64bit Ubuntu on a Linx 1010L Windows Tablet

After trying to follow several guides I found online, I finally managed to install and boot 64 bit Ubuntu, this will most likely work with other GRUB EFI based distros but hardware support may vary, here is my guide using the latest stable Ubuntu release (20.04). I will try to make it as easy to follow as possible, good luck!

A bit of backstory..

(Skip to the GUIDE If you just want to install)
So I was interested in getting a cheap, non-Android Tablet and stumbled across the Linx 1010L for a bargin.

Low specs but effective hardware for what it is:
• Intel Atom Z3735F (Bay-Trail) up to 1.8 GHz
• Intel HD Graphics (i915)
•  2GB LPDDR3 RAM
• 10.1 Inch Display
•  2x USB ports, 1 Mini HDMI, 1 SDcard slot, '32 GB' EMMC (More like 18 GB usable)

The issue comes with the 32 bit UEFI that is loaded on the device (and other Linx Windows Tablets) which causes brain ache because the CPU is 64 bit.. My guess is , realistically it is not a device made to run Windows 10 64 bit, so they ship it with the 32 bit UEFI to attempt to keep the device running what it is suited for.

But owners of this device will agree with me here, this device isn't even suited for Windows 10 32 bit. At all.

The device runs out of space too quickly, even with safely moving all the PATH variables to the SD card, there was't enough space to update Windows 10 from build 15xxx from 2015.
So I linked my activation to my account, and clean installed Windows 10 2004 onto the device..

Ran a bit better with all the optimizations to the OS over the years, but still the same kind of space issues coupled with having to manuall install the drivers for the Tablet (I backed them up with DISM)..
So I thought sod It, I will stick Linux on it, but nowadays 32 bit Linux is becoming less supported, most Distros have moved away from 32 bit almost completely. So I had to trawl the interwebs and couple together a few guides to get 64 bit (K)ubuntu installed.. Here's how I did it!

DO AT YOUR OWN RISK, IN THEORY YOUR WINDOWS ACTIVATION WILL BE SAVED AND YOU CAN GO BACK TO WINDOWS IF NEED BE, BUT BACK UP YOUR DATA!!

Before you begin, you'll need to get a few things:

• Ubuntu flavoured Distro: I use Kubuntu in this guide.
• USB Drive 4-8GB
• A USB keyboard or the Linx Keyboard
• 32 bit EFI file to load the 64 bit Linux Installer
• Rufus to write the ISO to USB (Others can be used)

• (OPTIONAL:) Backup your Windows Activation and download an OFFICIAL iso from the MS Website so you can go back to Windows if need be. (I will not cover this)

Part 1: Setting up the Live USB and Disabling Secure Boot.

First we need to burn the ISO to a usb drive (ALL DATA WILL BE LOST DURING BURNING) make sure you have the files needed. I will use Rufus as the Example

• Insert the USB stick, load up Rufus and select the ISO and start buring the image:

• Once that has finished successfully, open up the USB Drive in Explorer and copy the downloaded 32 bit EFI File to /EFI/BOOT/ on the USB Stick:

• Now Mine and a few others have Secure Boot already Disabled and USB Boot Enabled.. but to check this we need to turn on the Tablet HOLDING VOLUME+ AND POWER.
This will take us to the 'bios' menu.
Select 'Secure Boot Option' and turn Secure Boot Off:

(VOL UP + Power)
Tap on Secure Boot Options
Make Sure Secure Boot Is Disabled in Blue text, if not, Wipe Secure Boot Settings
Tap SCU to go to Bios Settings
Go to Boot - Make sure USB Booting is Enabled

Installing GRUB so we can actually boot

Now the Setup will fail at installing GRUB to the Tablet, I can't explain why, but it is to do with having a 32 bit EFI.
Next we will fix this and manually boot from the USB's GRUB into our fresh install on the Tablet


(Images are below to help)
• Boot back up off the USB
• When you reach GRUB, press 'c' on the Keyboard
This will drop to Grub Shell, now type these in order:
• set pager=1
• set root=(hd1,gpt3)
• linux /boot/vmlinuz-5.4.0-42-generic root=/dev/mmcblk1p3
• initrd /boot/initrd.img-5.4.0-42-generic
• boot

And now the Tablet should boot straight up from our newly Installed Drive!
Tip: if you use a different version of Ubuntu/Linux, when typing into grub, we can press TAB to 'autocomplete' file/folder names

boot

Now the tablet should be booting off the Tablet. We need to remove the Live USB as a source for APT and install GRUB.
Once you reach the desktop, connect to Wifi, open Konsole and type:

• sudo nano /etc/apt/sources.list

add a # in front of the line starting "deb cdrom:"
then press:

• Ctrl and X
• y
• Enter

This will block the live USB as a source, confirm changes to file and exit nano.
Now type into Konsole:

• sudo apt update && sudo apt install grub-efi-ia32

Accept the changes and wait for the process to finish, then type:

• sudo grub-install /dev/mmcblk1
•sudo update-grub

Now if all was input correct, GRUB will now be installed to the actual Tablet, to test, shut down and remove your USB, and power on! First boot will take a short while!

Notes and issues

Sadly there are a few issues I have noticed..

• The Auto-rotate doesn't work we have to switch rotation through System Settings in Ubutnu.
• When the screen is rotated, the touch axis doesn't change, so you will have mis-matched touches. I am currently looking for a fix for it.
• Both cameras don't work with Linux.
• Some users report Wifi issues, I haven't but speeds are not as fast as I had on Windows.
